Media Spend Analysis · Real Public Data

TL;DR — Analyzed real political committee disbursements from FEC bulk data files — converting pipe-delimited text into a cycle/month-partitioned Parquet warehouse and surfacing channel-level spend patterns across election cycles. Demonstrates a complete ETL workflow on genuine public data, with attention to schema design, query performance, and stakeholder-ready output.

Data source: FEC Bulk Data — operating expenditures and committee master files. Public domain, pipe-delimited. The full archive runs multi-GB; the fetch script (data/fetch_fec.py) pulls and joins the 2020/2022/2024 cycles, then writes the result as cycle/month-partitioned Parquet. The case study reads from there.

The Business Problem

Political ad spend is one of the few areas where real, granular media buying data is publicly available. Federal political committees must disclose every disbursement, including ad buys, to the FEC. The bulk data files contain millions of records spanning decades.

For a media measurement professional, this is gold, but the format is gnarly and very messy. Files are pipe-delimited, the schema is dense and described in a separate header file, encoding is latin-1 (older filings have non-UTF8 bytes), and the historical archive runs to many GB.

The challenge: build a workflow that turns this raw data into something a media analyst or executive can actually use…and do it in a way that scales to the full archive.

The Approach

1

Parse the FEC format

Pull the pipe-delimited bulk files from fec.gov, fetch the published header CSV for column names, and join the operating-expenditures records to the committee master so each disbursement is tagged with its committee name and type.

2

Design a partition strategy

Partition by election cycle (high-cardinality, queries naturally cycle-bounded) and month within cycle (mid-cardinality, supports time-series queries).

3

Categorize disbursements

Map free-text purpose descriptions to standardized spend categories (TV, digital, direct mail, etc.) using a deterministic rule-based classifier.

4

Surface analytical patterns

Aggregate to channel × cycle × committee-type level and produce executive-ready visualizations of spend allocation patterns.

Step 1: Load the FEC parquet cache

The fetch script (data/fetch_fec.py) downloads the operating expenditures and committee master zips from fec.gov, joins them on committee ID, and writes the result to cycle/month-partitioned Parquet. We read from there.

Step 2: Categorize free-text spend purposes

The FEC schema has a purpose field, but it’s free-text entered by the committee treasurers with no standardization. Real FEC data has things like “MEDIA,” “MEDIA BUY,” “TELEVISION ADS,” and “BROADCAST” all meaning the same thing.

A rule-based classifier handles this reliably without ML overhead:

Rule-based spend categorization 
import re

### categorization rules: (regex pattern, category)
### Order matters — more specific rules first
CATEGORY_RULES = [
    (r'(?<![A-Z0-9])(?:TV|T\.V\.|T\.V|TELEVISION|BROADCAST|CABLE|MEDIA\s+BUY)(?![A-Z0-9])','TV/Broadcast'),
    (r'(?<![A-Z0-9])(?:DIGITAL|ONLINE|SOCIAL\s+MEDIA|WEBSITE|FACEBOOK|GOOGLE)(?![A-Z0-9])', 'Digital'),
    (r'(?<![A-Z0-9])(?:DIRECT\s+MAIL|POSTAGE|PRINTING|MAIL\s+PRODUCTION)(?![A-Z0-9])', 'Direct Mail'),
    (r'(?<![A-Z0-9])(?:POLLING|SURVEY|FOCUS\s+GROUP|RESEARCH)(?![A-Z0-9])', 'Polling/Research'),
    (r'(?<![A-Z0-9])CONSULT\w*','Consulting'),
    (r'(?<![A-Z0-9])(?:PAYROLL|SALARY|OFFICE|RENT|TRAVEL|EVENT)(?![A-Z0-9])','Operations')]


def categorize(text):
    text = str(text).upper()

    for pattern, category in CATEGORY_RULES:
        if re.search(pattern, text):
            return category

    return 'Other'


oppexp_raw['category'] = oppexp_raw['purpose'].fillna('').apply(categorize)

## honest coverage numbers, the key results section reads these at render time
rec_cov = (oppexp_raw['category'] != 'Other').mean() * 100
named = oppexp_raw.loc[oppexp_raw['category'] != 'Other', 'transaction_amt'].sum()
dol_cov = named / oppexp_raw['transaction_amt'].sum() * 100

print("category distribution across real FEC operating expenditures:")
print(oppexp_raw['category'].value_counts().to_string())
print(f"\ncoverage: {rec_cov:.0f}% of records and {dol_cov:.0f}% of dollars get a named category")

print("\n example purpose strings the classifier saw (sample of 5 per category):")
for cat in ['TV/Broadcast', 'Digital', 'Direct Mail']:
    sample = (
        oppexp_raw[oppexp_raw['category'] == cat]
        ['purpose'].dropna().unique()[:5]
    )
    print(f"  {cat:<15} -> {list(sample)}")

Step 3: The partitioned Parquet warehouse

The right partitioning here is cycle (queries are almost always cycle-bounded) plus month within cycle (supports time-series analysis without too many tiny files). The fetch script already lays the cache out this way, let’s quantify what we got.

Step 4: Query with DuckDB

DuckDB on Parquet is the right tool here — full SQL, partition pruning, no infrastructure. Notice how queries that filter on cycle skip entire directories of files.

DuckDB analytical queries 
import duckdb

### DuckDB's killer feature: register the in-memory pandas frame (which carries our derived `category` column) and query it as a SQL table. Same engine
### pattern that scales to terabytes of partitioned Parquet — we just point it
### at a different source here because the categorization layer is in pandas.
con = duckdb.connect()
con.register("oppexp", oppexp_raw)

channel_by_cycle = con.sql("""
    SELECT
        cycle,
        category,
        SUM(transaction_amt) AS total_spend,
        COUNT(*) AS n_transactions,
        AVG(transaction_amt) AS avg_transaction
    FROM oppexp
    GROUP BY cycle, category
    ORDER BY cycle, total_spend DESC
""").df()

def fmt_currency(x):
    return f"${x:,.0f}"

display = channel_by_cycle.assign(
    total_spend     = channel_by_cycle["total_spend"].map(fmt_currency),
    avg_transaction = channel_by_cycle["avg_transaction"].map(fmt_currency),
    n_transactions  = channel_by_cycle["n_transactions"].map(lambda v: f"{v:,}"),
)
print("Channel spend by election cycle:")
print(display.to_string(index=False))

Top vendors by category — the kind of question stakeholders ask 
## light entity resolution before ranking. raw FEC names split the same firm
## into multiple rows: "RWT PRODUCTION LLC" vs "RWT PRODUCTION, LLC" vs
## "MEDIA BUYING & ANALYTICS" vs "MEDIA BUYING & ANALYTICS LLC"
### kill commas and periods, collapse space runs, drop trailing LLC / INC / etc,
### then title case. has to happen BEFORE the group by so the dollars combine
v = oppexp_raw['name'].fillna('').str.upper()
v = (v.str.replace(r'[.,]', ' ', regex=True)
      .str.replace(r'\s+', ' ', regex=True)
      .str.strip())
v = v.str.replace(r'(?:\s+(?:LLC|L L C|INC|LTD|LLP|CORP|CO|PC))+$', '', regex=True)
oppexp_raw['vendor_clean'] = v.str.title()

## re-register so duckdb sees the new column
con.register("oppexp", oppexp_raw)

top_vendors = con.sql("""
    SELECT
        category,
        vendor_clean AS vendor,
        SUM(transaction_amt) AS total_spend,
        COUNT(*) AS n_transactions
    FROM oppexp
    WHERE category IN ('TV/Broadcast', 'Digital', 'Direct Mail')
    GROUP BY category, vendor_clean
    QUALIFY ROW_NUMBER() OVER (PARTITION BY category ORDER BY total_spend DESC) <= 3
    ORDER BY category, total_spend DESC
""").df()

display_vendors = top_vendors.assign(
    total_spend    = top_vendors["total_spend"].map(fmt_currency),
    n_transactions = top_vendors["n_transactions"].map(lambda v: f"{v:,}"),
)
print("Top 3 vendors per major paid-media category:")

Step 5: Executive-ready visualization

Now the part stakeholders actually see. A chart is exec ready when the title states the finding, the labels sit on the data instead of in a legend box, and each chart makes exactly one point. Everything below follows those three rules.

Shared chart style and helpers 
import matplotlib.pyplot as plt
import matplotlib.ticker as mtick

## one palette for the whole piece. navy carries TV, blue is digital,
## slate fades mail into the background. red is reserved for election day
## so it always means the same thing
NAVY, BLUE, SLATE, RED, GRAY = '#0f172a', '#2563eb', '#94a3b8', '#dc2626', '#64748b'
colors = {'TV/Broadcast': NAVY, 'Digital': BLUE, 'Direct Mail': SLATE}

plt.rcParams.update({
    'axes.spines.top': False,
    'axes.spines.right': False,
    'figure.facecolor': 'white',
    'font.size': 10,
})

def fmt_dollars(v):
    ### $1.2B above a billion, $850M below
    return f'${v/1e9:.1f}B' if v >= 1e9 else f'${v/1e6:.0f}M'

def headline(ax, title, sub):
    ### takeaway goes big and bold, the literal description goes small and gray
    ax.set_title(title, fontsize=14, fontweight='bold', loc='left', pad=30, color=NAVY)
    ax.text(0, 1.06, sub, transform=ax.transAxes, fontsize=9.5, color=GRAY)

First view: the size of the pie. Every bar gets a dollar label so the y axis can go away entirely, nobody has to squint and interpolate.

How spend ramps to election day

A more interesting analytical view: how does spend ramp toward election day? The hockey stick is the expected part. The detail worth an annotation is what happens in election week itself, TV falls off a cliff while digital keeps buying right through it.

Weekly spend pattern leading up to election day 
# Compute days to election day for each cycle
election_days = {2020: '2020-11-03', 2022: '2022-11-08', 2024: '2024-11-05'}

oppexp_raw['election_day'] = oppexp_raw['cycle'].map(lambda c: pd.Timestamp(election_days[c]))
oppexp_raw['days_to_election'] = (oppexp_raw['election_day'] - oppexp_raw['transaction_dt']).dt.days
oppexp_raw['weeks_to_election'] = (oppexp_raw['days_to_election'] // 7)

## last 32 weeks before election, paid media only
recent = oppexp_raw[
    (oppexp_raw['weeks_to_election'].between(0, 32)) &
    (oppexp_raw['category'].isin(paid_media))
].copy()

weekly = (
    recent.groupby(['weeks_to_election', 'category'])['transaction_amt']
    .sum().reset_index()
)

## the number that earns the headline: how much of the 32 week window
## lands in the final 8 weeks
last8 = weekly.loc[weekly['weeks_to_election'] <= 8, 'transaction_amt'].sum()
pct_last8 = last8 / weekly['transaction_amt'].sum() * 100

fig, ax = plt.subplots(figsize=(8, 5))

## shade the home stretch so the eye goes there first
ax.axvspan(0, 8, color='#fef2f2', zorder=0)

for cat in paid_media:
    cat_data = weekly[weekly['category'] == cat].sort_values('weeks_to_election')
    ax.plot(cat_data['weeks_to_election'], cat_data['transaction_amt'] / 1e6,

ax.axvline(x=0, color=RED, linestyle='--', linewidth=2, alpha=0.7)

dig_final = weekly.loc[

Who actually gets the checks

What this demonstrates

Real public data, end-to-end. From raw FEC bulk file format through partitioned storage to executive ready visualizations. No synthetic shortcuts.

Pragmatic categorization. The free text purpose field is messy. Six rules name a category for 44% of records and 60% of dollars, and everything else stays in an honest Other bucket instead of getting force fitted. The rules have known rough edges too, the CABLE pattern pulls in some office utility bills alongside actual cable buys. The production path is widening the rule set against the biggest Other purposes by dollar volume, not reaching for an ML model.

Partition discipline. Cycle and month are the columns queries actually filter on. Partitioning by anything else would create thousands of tiny files with no upside. This goes to show to know you’re data and what you’re analyzing.

What I’d extend in production

  • Incremental ingestion — schedule weekly pulls of the latest filings rather than re-downloading everything (crontab, task scheduler, jenkins, js7, aws event bridge, google cloud scheduler)
  • Entity resolution on vendors — same media buyer often appears under slightly different names across filings (“AKPD MESSAGE & MEDIA” vs. “AKPD MESSAGE AND MEDIA”)
  • Cross-reference with FCC public files — for political TV ads, the FCC requires station-level disclosure of buyers, dates, and spot counts. Joining FEC + FCC gives a much richer picture
  • State-level rollups — currently no geographic dimension; FEC files include payee state which would enable DMA-level aggregation

## The Business Problem

Political ad spend is one of the few areas where **real, granular media buying data is
publicly available**. Federal political committees must disclose every disbursement,
including ad buys, to the FEC. The bulk data files contain millions of records spanning
decades.

For a media measurement professional, this is gold, but the format is gnarly and very messy. Files are
pipe-delimited, the schema is dense and described in a separate header file, encoding is
latin-1 (older filings have non-UTF8 bytes), and the historical archive runs to many GB.

The challenge: **build a workflow that turns this raw data into something a media analyst
or executive can actually use**...and do it in a way that scales to the full archive.

## Step 1: Load the FEC parquet cache

The fetch script (`data/fetch_fec.py`) downloads the operating expenditures and
committee master zips from `fec.gov`, joins them on committee ID, and writes the
result to cycle/month-partitioned Parquet. We read from there.

```{python}
#| label: load-fec-data
#| code-summary: "Load real FEC operating expenditures from Parquet cache"

from pathlib import Path
import numpy as np
import polars as pl
import pandas as pd

CACHE = Path("../data/cache/fec")
if not CACHE.exists() or not any(CACHE.rglob("*.parquet")):
    raise FileNotFoundError(
        "FEC cache not found. Run `python data/fetch_fec.py` from the project "
        "root first (downloads ~1 GB across the 2020/2022/2024 cycles)."
    )

oppexp_raw = (
    pl.scan_parquet(str(CACHE / "**/*.parquet"), hive_partitioning=True)
    .select([
        "cmte_id", "cmte_nm", "cmte_tp",
        "name", "transaction_dt", "transaction_amt",
        "purpose", "cycle",
    ])
    .collect()
    .to_pandas()
)
oppexp_raw["transaction_dt"] = pd.to_datetime(oppexp_raw["transaction_dt"])
oppexp_raw["transaction_amt"] = pd.to_numeric(oppexp_raw["transaction_amt"], errors="coerce")
oppexp_raw["cycle"] = pd.to_numeric(oppexp_raw["cycle"], errors="coerce").astype("Int64")

print(f"loaded {len(oppexp_raw):,} real expenditure records")
print(f"cycles: {sorted(oppexp_raw['cycle'].unique())}")
print(f"total spend: ${oppexp_raw['transaction_amt'].sum():,.0f}")
print(f"\nsample rows (FEC bulk format, post-join):")
print(oppexp_raw[['cmte_nm', 'name', 'transaction_dt', 'transaction_amt', 'purpose']]
      .head().to_string(index=False))
```

## Step 2: Categorize free-text spend purposes

The FEC schema has a `purpose` field, but it's free-text entered by the committee
treasurers with no standardization. Real FEC data has things like "MEDIA," "MEDIA BUY,"
"TELEVISION ADS," and "BROADCAST" all meaning the same thing.

A rule-based classifier handles this reliably without ML overhead:

```{python}
#| label: categorize-spend-cleaned
#| code-summary: "Rule-based spend categorization"

import re

### categorization rules: (regex pattern, category)
### Order matters — more specific rules first
CATEGORY_RULES = [
    (r'(?<![A-Z0-9])(?:TV|T\.V\.|T\.V|TELEVISION|BROADCAST|CABLE|MEDIA\s+BUY)(?![A-Z0-9])','TV/Broadcast'),
    (r'(?<![A-Z0-9])(?:DIGITAL|ONLINE|SOCIAL\s+MEDIA|WEBSITE|FACEBOOK|GOOGLE)(?![A-Z0-9])', 'Digital'),
    (r'(?<![A-Z0-9])(?:DIRECT\s+MAIL|POSTAGE|PRINTING|MAIL\s+PRODUCTION)(?![A-Z0-9])', 'Direct Mail'),
    (r'(?<![A-Z0-9])(?:POLLING|SURVEY|FOCUS\s+GROUP|RESEARCH)(?![A-Z0-9])', 'Polling/Research'),
    (r'(?<![A-Z0-9])CONSULT\w*','Consulting'),
    (r'(?<![A-Z0-9])(?:PAYROLL|SALARY|OFFICE|RENT|TRAVEL|EVENT)(?![A-Z0-9])','Operations')]


def categorize(text):
    text = str(text).upper()

    for pattern, category in CATEGORY_RULES:
        if re.search(pattern, text):
            return category

    return 'Other'


oppexp_raw['category'] = oppexp_raw['purpose'].fillna('').apply(categorize)

## honest coverage numbers, the key results section reads these at render time
rec_cov = (oppexp_raw['category'] != 'Other').mean() * 100
named = oppexp_raw.loc[oppexp_raw['category'] != 'Other', 'transaction_amt'].sum()
dol_cov = named / oppexp_raw['transaction_amt'].sum() * 100

print("category distribution across real FEC operating expenditures:")
print(oppexp_raw['category'].value_counts().to_string())
print(f"\ncoverage: {rec_cov:.0f}% of records and {dol_cov:.0f}% of dollars get a named category")

print("\n example purpose strings the classifier saw (sample of 5 per category):")
for cat in ['TV/Broadcast', 'Digital', 'Direct Mail']:
    sample = (
        oppexp_raw[oppexp_raw['category'] == cat]
        ['purpose'].dropna().unique()[:5]
    )
    print(f"  {cat:<15} -> {list(sample)}")
```

## Step 4: Query with DuckDB

DuckDB on Parquet is the right tool here — full SQL, partition pruning, no infrastructure.
Notice how queries that filter on `cycle` skip entire directories of files.

```{python}
#| label: duckdb-queries
#| code-summary: "DuckDB analytical queries"

import duckdb

### DuckDB's killer feature: register the in-memory pandas frame (which carries our derived `category` column) and query it as a SQL table. Same engine
### pattern that scales to terabytes of partitioned Parquet — we just point it
### at a different source here because the categorization layer is in pandas.
con = duckdb.connect()
con.register("oppexp", oppexp_raw)

channel_by_cycle = con.sql("""
    SELECT
        cycle,
        category,
        SUM(transaction_amt) AS total_spend,
        COUNT(*) AS n_transactions,
        AVG(transaction_amt) AS avg_transaction
    FROM oppexp
    GROUP BY cycle, category
    ORDER BY cycle, total_spend DESC
""").df()

def fmt_currency(x):
    return f"${x:,.0f}"

display = channel_by_cycle.assign(
    total_spend     = channel_by_cycle["total_spend"].map(fmt_currency),
    avg_transaction = channel_by_cycle["avg_transaction"].map(fmt_currency),
    n_transactions  = channel_by_cycle["n_transactions"].map(lambda v: f"{v:,}"),
)
print("Channel spend by election cycle:")
print(display.to_string(index=False))
```

```{python}
#| label: top-vendors
#| code-summary: "Top vendors by category — the kind of question stakeholders ask"

## light entity resolution before ranking. raw FEC names split the same firm
## into multiple rows: "RWT PRODUCTION LLC" vs "RWT PRODUCTION, LLC" vs
## "MEDIA BUYING & ANALYTICS" vs "MEDIA BUYING & ANALYTICS LLC"
### kill commas and periods, collapse space runs, drop trailing LLC / INC / etc,
### then title case. has to happen BEFORE the group by so the dollars combine
v = oppexp_raw['name'].fillna('').str.upper()
v = (v.str.replace(r'[.,]', ' ', regex=True)
      .str.replace(r'\s+', ' ', regex=True)
      .str.strip())
v = v.str.replace(r'(?:\s+(?:LLC|L L C|INC|LTD|LLP|CORP|CO|PC))+$', '', regex=True)
oppexp_raw['vendor_clean'] = v.str.title()

## re-register so duckdb sees the new column
con.register("oppexp", oppexp_raw)

top_vendors = con.sql("""
    SELECT
        category,
        vendor_clean AS vendor,
        SUM(transaction_amt) AS total_spend,
        COUNT(*) AS n_transactions
    FROM oppexp
    WHERE category IN ('TV/Broadcast', 'Digital', 'Direct Mail')
    GROUP BY category, vendor_clean
    QUALIFY ROW_NUMBER() OVER (PARTITION BY category ORDER BY total_spend DESC) <= 3
    ORDER BY category, total_spend DESC
""").df()

display_vendors = top_vendors.assign(
    total_spend    = top_vendors["total_spend"].map(fmt_currency),
    n_transactions = top_vendors["n_transactions"].map(lambda v: f"{v:,}"),
)
print("Top 3 vendors per major paid-media category:")
print(display_vendors.to_string(index=False))
```

Production Llc" and "Rwt Production, Llc" showed up as two separate vendors, split by
nothing but a comma, and Media Buying & Analytics appeared with and without its "LLC."
The normalization in Step 4 (strip punctuation, collapse spacing, drop corporate
suffixes) merges those rows before ranking, which changes the standings. Harder cases
like "AND" vs "&" still need the fuzzy matching described in the production section.

## What this demonstrates

**Real public data, end-to-end.** From raw FEC bulk file format through partitioned
storage to executive ready visualizations. No synthetic shortcuts.

**Pragmatic categorization.** The free text purpose field is messy. Six rules name a
category for `{python} f"{rec_cov:.0f}%"` of records and `{python} f"{dol_cov:.0f}%"` of
dollars, and everything else stays in an honest Other bucket instead of getting force
fitted. The rules have known rough edges too, the CABLE pattern pulls in some office
utility bills alongside actual cable buys. The production path is widening the rule set
against the biggest Other purposes by dollar volume, not reaching for an ML model.

**Partition discipline.** Cycle and month are the columns queries actually filter on.
Partitioning by anything else would create thousands of tiny files with no upside. This goes to show to know you're data and what you're analyzing. 

## What I'd extend in production

- **Incremental ingestion** — schedule weekly pulls of the latest filings rather than
  re-downloading everything (crontab, task scheduler, jenkins, js7, aws event bridge, google cloud scheduler)
- **Entity resolution on vendors** — same media buyer often appears under slightly different
  names across filings ("AKPD MESSAGE & MEDIA" vs. "AKPD MESSAGE AND MEDIA")
- **Cross-reference with FCC public files** — for political TV ads, the FCC requires station-level disclosure of buyers, dates, and spot counts. Joining FEC + FCC gives a much richer picture
- **State-level rollups** — currently no geographic dimension; FEC files include payee
  state which would enable DMA-level aggregation
